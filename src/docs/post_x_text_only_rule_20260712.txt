post_x 運用ルール案

1. 日々のツイートにはURLを載せない
2. 日々のツイートはreplyしない
3. マガジンへの導線はプロフィールと固定ツイートに置く
4. 日々のツイートは、記事をランダムで1件選び、要約して投稿する
5. 要約は有料1までの入口情報にとどめる
6. 続きが気になる形で止める
7. 1日1投稿でも運用できるように単発完結にする

狙い

- reply周りの制約を避ける
- 投稿コストを下げる
- 文章品質の調整をしやすくする
- 固定ツイートとプロフィールに導線を集約する

注意点

- 投稿からの直接クリックは減る
- 固定ツイートとプロフィールの整備が重要になる
- 要約を出しすぎると販促力が落ちる
