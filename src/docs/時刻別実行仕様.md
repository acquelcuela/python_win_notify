# NightlyBatchNotify 時刻別実行仕様

## 概要

NightlyBatchNotify は、Windows のタスクスケジューラから 15 分ごとに起動されるバッチです。
実際にどの処理を動かすかは `config.json` で制御します。

この版では、重い処理を毎回回さず、時刻ごとにモジュールを分ける前提にしています。

## 実行時刻

推奨の実行時刻は次の 4 つです。

```json
{
  "batch_schedule": ["07:00", "09:30", "12:15", "22:45"],
  "batch_window_minutes": 14
}
```

- `07:00` 朝の総点検
- `09:30` 寄り付き後の確認
- `12:15` 午前ニュースと午前相場のまとめ
- `22:45` 夜の市場データ確認

## 時刻ごとのモジュール

### 07:00

- `stock_nikkei`
- `stock_watchlist`
- `stock_dividend`
- `market_news`
- `news_movers`
- `stock_x_trends`
- `ai_summary`
- `report_html`
- `mail_gmail`

### 09:30

- `stock_nikkei`
- `stock_watchlist`
- `market_news`
- `news_movers`
- `ai_summary`
- `report_html`
- `mail_gmail`

### 12:15

- `stock_nikkei`
- `stock_watchlist`
- `market_news`
- `news_movers`
- `ai_summary`
- `report_html`
- `mail_gmail`

### 22:45

- `stock_nikkei`
- `stock_watchlist`
- `stock_dividend`
- `report_html`
- `mail_gmail`

## 方針

- ニュース取得と AI 要約は 1 日 3 回までに抑える
- `22:45` は市場データ中心にして、RSS や Grok / Gemini の呼び出しを避ける
- `stock_x_trends` は 07:00 のみで実行する
- `post_x_note` は別ランナーで、現在は OFF のままにする

## モジュール概要

### `stock_nikkei`

日経平均、日経225先物、TOPIX 連動 ETF を取得します。

### `stock_watchlist`

注目銘柄の前日比や数日分の変化を取得します。

### `stock_dividend`

RYLD / SDIV の配当タイミングを扱います。

### `market_news`

日本株関連のニュースを RSS から取得します。

### `news_movers`

ニュース本文から銘柄を照合し、値動き銘柄を作ります。

### `stock_x_trends`

Grok を使って、日本株に関係する X 上の話題銘柄やテーマを拾います。

### `ai_summary`

市場データとニュースを AI で要約します。

### `report_html`

各モジュールの出力 JSON を HTML にまとめます。

### `mail_gmail`

HTML レポートを Gmail で送信します。

### `post_x_note`

note 記事を X に投稿する独立ランナーです。

## 必要な環境変数

`post_x_note` を使う場合:

```env
X_API_KEY=...
X_API_SECRET=...
X_ACCESS_TOKEN=...
X_ACCESS_TOKEN_SECRET=...
GEMINI_API_KEY=...
```

`stock_x_trends` を使う場合:

```env
GROK_API_KEY=...
```

## 補足

- `main.py` は時刻枠に応じて `schedule_modules` の内容を実行します
- `run_force.bat` はテスト用に時刻枠を選べます
- `stock_x_trends` は 07:00 にだけ回すことで、Grok の呼び出しを 1 日 1 回に抑えます
