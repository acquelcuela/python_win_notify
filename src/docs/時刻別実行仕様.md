# NightlyBatchNotify 時刻別実行仕様

最終更新: 2026-07-14

## 概要

NightlyBatchNotify は、Windows のタスクスケジューラから 15 分ごとに起動されるバッチです。
`main.py` が現在時刻を見て、`config.json` の `batch_schedule` に定義された時刻の
`±batch_window_minutes`(既定14分)以内かどうかを判定し、対象時刻内であれば処理を実行します
(それ以外は何もせず終了)。

タスクスケジューラ自体は登録時刻(0:00)から1日中15分おきに動き続けるため、
`batch_schedule` に時刻を追加・削除するだけで実行タイミングを変更でき、
タスクスケジューラの再登録は不要です(`scheduler\install_scheduled_task.ps1` は
`config.json` の `batch_schedule` を毎回読み直して登録するので、時刻を大きく作り直す
場合のみ再実行すれば十分です)。

## 実行時刻

```json
{
  "batch_schedule": ["07:00", "08:00", "09:30", "12:15", "21:00", "22:45"],
  "batch_window_minutes": 14,
  "batch_weekdays_only": true
}
```

- `07:00` 朝の総点検(市況・ニュース・Xトレンド検索・メイン配信)
- `08:00` post_x_magazine 専用枠(朝)
- `09:30` 寄り付き後の確認
- `12:15` 午前ニュースと午前相場のまとめ
- `21:00` post_x_magazine 専用枠(夜)
- `22:45` 夜の市場データ確認

`batch_weekdays_only: true` のため、土日は全時刻がスキップされます。

## 時刻ごとのモジュール

`config.json` の `schedule_modules` で明示的に制限されている時刻は、そこに書かれた
モジュールだけが動きます。制限されていない時刻は、`modules` で有効化されている
モジュールが**全て**動きます。

```json
"schedule_modules": {
  "08:00": ["post_x_magazine"],
  "21:00": ["post_x_magazine"]
}
```

### 07:00 / 09:30 / 12:15 / 22:45(制限なし・共通)

`modules` で `true` になっている全モジュールが毎回動きます:

- `stock_nikkei`
- `stock_watchlist`
- `stock_dividend`
- `market_news`
- `news_movers`
- `stock_x_trends`(実際に検索するのは07:00のみ。他の時刻は後述の理由でスキップ)
- `ai_summary`
- `post_x_magazine`(実際に投稿するのは08:00/21:00のみ。他の時刻は`run_times`で自己スキップ)
- `report_html`
- `mail_gmail`

> 補足: 以前のドキュメントでは「09:30はニュース系を省く」「22:45は市場データのみ」という
> 軽量構成を想定していましたが、現在の `config.json` の `schedule_modules` にはその制限が
> 定義されておらず、実際には4回とも同じフルセットのモジュールが動いています。
> 軽量化したい場合は `schedule_modules` に `09:30`/`12:15`/`22:45` の制限リストを追加する
> 必要があります(未実施)。

### 08:00 / 21:00(post_x_magazineのみ)

`post_x_magazine` だけが動きます。市況データ取得・ニュース取得・レポート生成・
メイン配信メールは一切走りません(post_x_magazine自身が結果を専用の通知メールで送ります)。

## モジュールが内部で持つ時刻ゲート

`schedule_modules` とは別に、モジュール自身が `BATCH_SCHEDULE_KEY` 環境変数を見て
「今回は実処理をするか」を自己判定しているものがあります。

### `stock_x_trends`

```json
"stock_x_trends": {
  "enabled": true,
  "model": "grok-4.3",
  "max_tokens": 1000,
  "run_times": ["07:00"]
}
```

- Grok への実検索は `run_times`(既定 `["07:00"]`)に一致する時刻のみ実行
- それ以外の時刻はスキップし、`output/stock_x_trends.json` を**上書きしない**
  (朝の検索結果がそのまま残る)
- `report_html.py` はこのファイルの中身をそのまま表示するため、09:30/12:15/22:45の
  メールにも朝のXトレンド結果が引き続き掲載される
- これにより Grok API 呼び出しは1日1回に抑えられる

### `post_x_magazine`

```json
"post_x_magazine": {
  "run_times": ["08:00", "21:00"],
  ...
}
```

- `run_times` に一致しない時刻は「schedule not allowed」で即スキップ(投稿・メールなし)
- 08:00/21:00 では実際にXへ投稿し、成功・失敗いずれでも専用の通知メールを送信

## モジュール概要

### `stock_nikkei`

日経平均、日経225先物、TOPIX 連動 ETF を取得します。

### `stock_watchlist`

注目銘柄の前日比と、直近10営業日分の日次変化を取得します
(yfinanceから15営業日分取得し、直近11本の終値から10日分の前日比を算出)。

### `stock_dividend`

RYLD / SDIV の配当タイミングを扱います。

### `market_news`

日本株関連のニュースを Google News RSS(Google News / 株探 / Reuters の3系統)から
取得します。

### `news_movers`

ニュース本文から銘柄を照合し、値動き銘柄を作ります。

### `stock_x_trends`

Grok を使って、日本株に関係する X 上の話題銘柄やテーマを拾います。実検索は07:00のみ。

### `ai_summary`

市場データとニュースを AI で要約します。

### `post_x_magazine`

`config.json` の `post_x_magazine.magazines` に登録されたテーマ別マガジンから記事を
ローテーション選択し、Gemini(失敗時は本文ベースのフォールバック)で5本のXスレッドを
生成して投稿します。

- 1本目: フック(数字・体験談ベースの書き出し)+ 導入
- 2本目: 内容の具体点
- 3本目: 実用性・気づき
- 4本目: 読後の持ち帰り + 読者への問いかけ(返信を誘う)
- 5本目: マガジン紹介(他のテーマ別マガジンをプロフィール固定ツイートで案内)+ ハッシュタグ

`POST_X_MAGAZINE_PREVIEW=1` / `POST_X_MAGAZINE_NOTIFY_ONLY=1` を立てるとXには投稿せず、
生成結果のみメール通知できます。成功・失敗いずれの場合も通知メールが送られ、本文には
5投稿すべてが並び、失敗時はどの投稿で失敗したか(`n/5`)と理由が表示されます。

手動で単体実行したい場合は `run_post_x_magazine.bat`(`main.py --force --schedule 08:00`
を実行し、`schedule_modules` により post_x_magazine のみが動く)を使います。

### `report_html`

各モジュールの出力 JSON を HTML にまとめます。

### `mail_gmail`

HTML レポートを Gmail で送信します。`post_x_magazine` の通知メールもこのモジュール内の
`send_html_mail()` を共有して送信します。

### `post_x_note`

note 記事を X に投稿する旧世代の独立ランナーです。`post_x_magazine` に役割を引き継いだため
`modules` には登録されておらず、現在は常時OFFです(`run_post_x_note.bat` から手動実行は可能)。

## 必要な環境変数

```env
GMAIL_ADDRESS=...
GMAIL_APP_PASSWORD=...
MAIL_TO=...
GEMINI_API_KEY=...
GROK_API_KEY=...

# post_x_magazine / post_x_note でXに投稿する場合のみ必要
X_API_KEY=...
X_API_SECRET=...
X_ACCESS_TOKEN=...
X_ACCESS_TOKEN_SECRET=...
```

## 補足

- `main.py` は `--force --schedule HH:MM` で任意の時刻枠を強制実行できます(時刻チェック省略)
- `run_force.bat` はテスト用に07:00/09:30/12:15/22:45または全モジュールを選べる対話式バッチです
  (08:00/21:00枠は未対応。post_x_magazine単体は `run_post_x_magazine.bat` を使ってください)
