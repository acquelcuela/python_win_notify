# noteマガジン返信運用 仕様メモ 2026-07-10

## 目的
note の記事を、あらかじめ用意した「マガジン用の種ツイート」に対する返信として宣伝する。

運用の狙いは次のとおり。

- note 記事の導線を毎日安定して出す
- 投稿ネタをマガジン単位で輪番化し、宣伝の偏りを減らす
- 1回の実行で AI による文面生成を行い、作業を自動化する
- まずは 1 段階のみの返信投稿で開始する

## 前提

- 実行は 1 日 2 回とする
- 各実行で 1 件だけ投稿する
- 投稿形式は「マガジン用の種ツイートへの返信」
- 返信本文は AI で生成する
- 画像添付やスレッド化は当面扱わない
- まずは note 記事 1 本に対する 1 回の返信だけを対象とする

## 用語

- 種ツイート: note のマガジンを代表する固定の親投稿
- マガジン: 記事の分類単位
- 返信投稿: 種ツイートにぶら下げる宣伝文

## 要件

### 機能要件

1. マガジンごとに種ツイートを登録できること
2. 実行時に、登録済みマガジンを輪番で 1 件選択できること
3. 選ばれたマガジンに紐づく note 記事を 1 件選ぶこと
4. AI が返信文を 1 件生成すること
5. 生成文を指定の種ツイートへ返信として投稿すること
6. 1 日 2 回の定期実行に対応すること
7. 投稿済み記事の重複を避けること
8. 失敗時は記録を残し、次回実行に影響を残しすぎないこと

### 非機能要件

- 返信文は短く、スマホで読みやすいこと
- 種ツイートとの関係が分かること
- AI 生成コストを抑えること
- 再実行しても同じ記事を出し続けないこと
- Windows 上の既存バッチ運用に乗ること

## 仕様

### マガジン設定

設定ファイルに、マガジンの一覧を持つ。

各マガジンは次の情報を持つ。

- `id`: 識別子
- `name`: 表示名
- `tweet_id`: 種ツイートの X 投稿ID
- `url`: 種ツイートの URL
- `article_keys`: このマガジンに属する note 記事の key 一覧
- `enabled`: 有効/無効

### 輪番選択

- 実行ごとに、前回使ったマガジンの次を選ぶ
- 末尾まで来たら先頭に戻る
- 無効なマガジンは飛ばす
- 種ツイートが未登録のマガジンも飛ばす

### 記事選択

- 選ばれたマガジンに属する記事から候補を作る
- 既に投稿済みの記事は除外する
- 除外キー、除外 URL、履歴日数の判定は既存の `post_x_note` と同様に扱う
- 候補がなければその回はスキップする

### AI 生成

- AI は返信文だけを 1 件生成する
- 1 段階のみとする
- 返信本文は短めにする
- URL は含めてもよいが、X 側の制約で通らない場合は代替文面を使う
- 返信文の冒頭は不自然な見出しにしない
- ハッシュタグの有無は設定で制御できるようにする

### 投稿

- まず種ツイートに対する reply として投稿する
- 失敗した場合はログに残す
- 必要なら、後続で「URLなし版」などの再試行を行う

### 実行頻度

- 1 日 2 回
- 各回 1 投稿
- 実行時刻は既存のバッチスケジュールに従う

### 状態管理

次の情報を保存する。

- 最後に使ったマガジン ID
- 最後に使った記事 key
- 投稿成功時刻
- 失敗理由
- 再試行の有無

## 設定案

`config.json` には次のような設定を置く想定。

```json
{
  "post_x_magazine": {
    "enabled": false,
    "model": "gemini-3.1-flash-lite",
    "run_times_per_day": 2,
    "rotation_state_file": "state/post_x_magazine_rotation.json",
    "posted_history_file": "state/post_x_magazine_history.json",
    "magazines": [
      {
        "id": "high_dividend_review",
        "name": "高配当株の個別レビュー",
        "tweet_id": "1234567890",
        "url": "https://x.com/...",
        "article_keys": ["n0ca2d1025c7b", "naf1745393ea1"],
        "enabled": true
      }
    ]
  }
}
```

## 実行フロー

1. 実行時刻に到達
2. 輪番状態を読む
3. 次のマガジンを選ぶ
4. そのマガジンの記事候補を抽出する
5. 返信文を AI で生成する
6. 種ツイートへ reply する
7. 結果を履歴に保存する
8. 次回用の輪番状態を更新する

## 失敗時の扱い

- 種ツイート未設定: スキップ
- 記事候補なし: スキップ
- AI 生成失敗: ログを残して終了
- reply 投稿失敗: ログを残して終了
- 同一記事の重複: 除外して別記事にする

## 今回の段階

最初の段階では次だけを実装対象にする。

- マガジン設定
- 輪番選択
- AI による 1 件返信文生成
- 1 日 2 回の実行

以下は後回しにする。

- 1 マガジンに複数記事を束ねる
- 連続スレッド化
- 種ツイートの自動生成
- 画像付き投稿
- 反応計測
