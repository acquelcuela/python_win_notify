# post_x note 日次投稿仕様 2026-07-12

## 目的
note の内容をもとに、X へ日次で投稿する。
宣伝よりも、読んだ人がすぐ使える情報を優先する。

## ルール

- 日本語で書く
- URL は本文に入れない
- reply 先を示す文言は入れない
- note の内容は、肝の数字や結論を前面に出す
- 有料部分が読める場合は、無理に入れず、品質向上に役立つ範囲だけ取り込む
- 記事・宣伝・集客の説明を前面に出しすぎない
- 1 本目の最初の行は必ず【タイトル】にする
- 1 本目から 4 本目までのスレッド形式で投稿する
- 1 本あたり 5 行を目安にする
- 1 本あたり 140 文字前後、最大でも 180 文字以内を目標にする
- ハッシュタグは最後の投稿にだけ付ける
- ハッシュタグは 4〜6 個にする

## 投稿の考え方

- 1 本目: 題材と結論を先に出す
- 2 本目: 数字や比較条件を出す
- 3 本目: その数字が意味することを出す
- 4 本目: 使いどころや読み方をまとめる

## 出し方の優先順位

1. 結論や差が大きい条件
2. 年利、生活費、共働き・単身などの比較軸
3. 数字の意味
4. 実生活での使い方

## 文章のトーン

- 抽象的な感想だけで終わらせない
- 本文の具体的な数字や条件を含める
- 読んだだけで比較の軸が分かるようにする
- 断定しすぎず、でも中身ははっきり見せる

## 状態メモ

- 有料部分は必須ではない
- ただし、読めるなら内容の質を上げるために使ってよい
- 投稿は宣伝文ではなく、情報として読める形を優先する

## 実装のポイント

- ランダム選択した note をもとに 4 本のスレッドを生成する
- 取得できた本文から要約する
- 結論や数字が弱い場合は、本文の具体文を優先する
- 最後の投稿のみハッシュタグを付ける
